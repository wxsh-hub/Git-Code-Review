## 1. 环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| JDK | 1.8+ | 推荐 Oracle JDK 8 / OpenJDK 8 / Azul Zulu 8 |
| Maven | 3.6+ | 用于编译构建（开发模式必需，生产模式可选） |
| 内存 | 1 GB+ | JVM 启动参数默认为 `-Xms1g -Xmx1g` |

验证环境：

```bash
java -version
# 应输出: java version "1.8.0_xxx" 或更高

mvn -version
# 应输出: Apache Maven 3.6.x 或更高
```

## 2. 从源码启动（开发模式）

> 适用场景：本地开发、调试、不想打包

### 2.1 进入项目目录

```bash
cd devops-ai
```

### 2.2 编译并启动

```bash
mvn clean package -pl devops-ai-bootstrap -am -DskipTests
java -Xms1g -Xmx1g -Dfile.encoding=UTF-8 -jar devops-ai-bootstrap/target/devops-ai-bootstrap-1.0.0.jar
```

或者用 Spring Boot Maven 插件直接运行：

```bash
mvn spring-boot:run -pl devops-ai-bootstrap
```

### 2.3 启动成功标志

控制台输出类似以下内容即为启动成功：

```
=============================================
  devops-ai 文档生成服务 v1.0.0
  启动中...
=============================================
...
Tomcat started on port(s): 8070 (http)
...
Started DevOpsAiApplication in X.XXX seconds
```

## 3. 打包部署启动（生产模式）

> 适用场景：服务器部署、分发交付

### 3.1 构建完整安装包

```bash
cd devops-ai
mvn clean package -DskipTests
```

构建完成后，安装包位于：

```
devops-ai-bootstrap/target/devops-ai-bootstrap-1.0.0-bin.zip
```

### 3.2 解压并启动

```bash
# 解压
unzip devops-ai-bootstrap/target/devops-ai-bootstrap-1.0.0-bin.zip
cd devops-ai-v1.0.0

# ── Windows ──
start.bat                  # 前台运行
start.bat --daemon         # 后台运行

# ── Linux / Mac ──
chmod +x start.sh
./start.sh                 # 前台运行（默认）
./start.sh start           # 后台启动
./start.sh stop            # 停止服务
./start.sh restart         # 重启服务
./start.sh status          # 查看运行状态
```

### 3.3 安装包目录结构

```
devops-ai-v1.0.0/
├── devops-ai-bootstrap-1.0.0.jar    # 主程序 JAR
├── config/
│   └── application.yml              # 配置文件（可按需修改）
├── start.bat                         # Windows 启动脚本
├── start.sh                          # Linux/Mac 启动脚本
├── 用户使用手册.md
├── 用户使用手册.html
├── data/                             # 运行时自动创建：H2 数据库文件
├── output/                           # 运行时自动创建：输出文档目录
└── logs/                             # 运行时自动创建：日志目录
```

## 5. 启动脚本说明

### 5.1 Windows (`start.bat`)

| 命令 | 说明 |
|------|------|
| `start.bat` | 前台启动，关闭窗口即停止 |
| `start.bat --daemon` 或 `start.bat -d` | 后台启动，需用任务管理器停止 |

### 5.2 Linux / Mac (`start.sh`)

| 命令 | 说明 |
|------|------|
| `./start.sh` | 前台启动，Ctrl+C 停止 |
| `./start.sh start` | 后台启动，写入 `.app.pid` 文件 |
| `./start.sh stop` | 优雅停止（SIGTERM → 超时 2s → SIGKILL） |
| `./start.sh restart` | 先 stop 再 start |
| `./start.sh status` | 检查进程是否存活 |

## 6. 配置说明

核心配置文件：`devops-ai-bootstrap/src/main/resources/application.yml`（源码）或 `config/application.yml`（部署包）

### 6.1 关键配置项

```yaml
server:
  port: 8070                            # 服务端口

spring:
  datasource:
    url: jdbc:h2:file:./data/devops-ai  # H2 数据库文件路径

devops:
  ai:
    encrypt:
      key: devops-ai-secret-key-2026    # 加密密钥（敏感配置加密用）
    generator:
      output-dir: ./output              # 文档输出目录
    cache:
      project-ttl-minutes: 60           # 项目缓存时间（分钟）
      branch-ttl-minutes: 5             # 分支缓存时间（分钟）
    task:
      max-concurrent: 5                 # 最大并发任务数

ocr:
  bin:
    path: "./bin/ocr.exe"              # OCR 二进制路径（代码审查功能，可选）
  fallback-on-error: true              # OCR 失败时是否回退到 AI 审查
```

### 6.2 修改端口

编辑 `application.yml`：

```yaml
server:
  port: 9090   # 改为你需要的端口
```

### 6.3 修改 JVM 内存

编辑 `start.bat` / `start.sh` 中的 `-Xms1g -Xmx1g` 参数：

```bash
# 改为 2G
java -Xms2g -Xmx2g -jar ...
```

## 7. 常见问题

### Q1: 端口被占用

**错误**：`Port 8070 is already in use` 或 `bind: address already in use`

**解决**：
```bash
# 查找占用进程
# Windows
netstat -ano | findstr 8070

# Linux/Mac
lsof -i :8070

# 然后修改 application.yml 中的 server.port，或关闭占用进程
```

### Q2: 找不到 Java

**错误**：`[错误] 未找到 Java 运行环境`

**解决**：安装 JDK 8+，并确保 `java` 加入系统 PATH 环境变量。

### Q3: Maven 编译失败

**错误**：`Could not resolve dependencies`

**解决**：
```bash
# 清除本地缓存重试
mvn clean package -DskipTests -U
```

### Q4: H2 数据库文件锁定

**错误**：`Database may be already in use`

**解决**：确保只有一个 devops-ai 实例在运行。如需重置数据，删除 `data/` 目录下所有文件后重启。

### Q5: 启动后页面无法访问

1. 确认控制台日志中 `Tomcat started on port(s): 8070` 已输出
2. 检查防火墙是否开放了对应端口
3. 确认浏览器代理设置没有拦截 localhost

### Q6: OCR 功能报错

如果不需要 OCR 代码审查功能，可以直接忽略 OCR 相关错误。或者修改配置：

```yaml
ocr:
  fallback-on-error: true   # 已默认开启，出错会自动回退到 AI 审查
```

