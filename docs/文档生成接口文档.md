# 文档生成服务 - 第三方对接接口文档

**基础地址**: `https://<your-host>/api/v1`
**Content-Type**: `application/json`  


---

## 一、提交文档生成任务

### GET /api/v1/generate（快速提交）

查询参数方式提交，支持同步/异步两种模式。

**查询参数**: 额外增加 `sync` 参数。

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `projectCode` | **是** | 项目编码 | - |
| `branch` | 否 | Git分支 | 项目默认分支 |
| `templateName` | 否 | 文档模板名 | 项目默认模板 |
| `format` | 否 | 输出格式 | `markdown` |
| `projectVersion` | 否 | 版本号 | - |
| `author` | 否 | 作者筛选 | - |
| `sinceHash` | 否 | 起始 commit hash | - |
| `incremental` | 否 | 增量模式 | `false` |
| `useAiClassifier` | 否 | AI 分类 | `false` |
| `sync` | 否 | 是否同步模式 | `false` |
| `useCodeReview` | 否 | 代码审查 | `false` |
| `reviewFormat` | 否 | 审查文档格式 | `markdown` |

**异步模式**（`sync=false`，默认）：返回 `taskId`。

**同步模式**（`sync=true`）：轮询等待最多 120 秒，任务完成后直接返回文件下载，不返回 JSON。

## 二、查询任务状态

### GET /api/v1/tasks/{taskId}

**响应示例**:
```json
{
  "success": true,
  "data": {
    "taskId": "a1b2c3d4-...",
    "status": "completed",
    "progress": 100,
    "createdAt": "2026-05-04T10:00:00",
    "completedAt": "2026-05-04T10:00:05",
    "format": "markdown",
    "outputPath": "/output/.../CHANGELOG.md",
    "errorMessage": null
  }
}
```

**status 说明**:

| 状态 | 说明 |
|------|------|
| `pending` | 排队等待处理 |
| `running` | 正在生成文档 |
| `completed` | 生成成功，可下载文件 |
| `failed` | 生成失败，查看 `errorMessage` |

---

## 三、获取生成结果

### GET /api/v1/tasks/{taskId}/download（推荐）

以文件流方式下载生成的文档，响应头包含 `Content-Disposition: attachment`。

- 格式为 `markdown` 时，Content-Type: `text/markdown`
- 格式为 `html` 时，Content-Type: `text/html`

### GET /api/v1/tasks/{taskId}/result

以文本方式返回生成的文档内容，可直接在页面展示或二次处理。不支持时返回 HTTP 409。

---

## 四、建议对接流程

### 异步模式（推荐）

```
① GET /api/v1/generate?projectCode=xxx&...        → 提交任务，获取 taskId
② GET /api/v1/tasks/{taskId}                       → 每 2-3 秒轮询状态直到 completed
③ GET /api/v1/tasks/{taskId}/download              → 下载最终文档
```

### 同步模式（适合快速调用）

```
① GET /api/v1/generate?projectCode=xxx&sync=true   → 等待最多 120 秒，直接返回文件
```

## 五、错误码

| HTTP 状态码 | 说明 |
|-------------|------|
| 200 | 成功 |
| 400 | 参数错误（projectCode 为空、时间与 hash 范围冲突、项目编码不存在等） |
| 404 | 任务不存在 |
| 408 | 同步模式超时（超过 120 秒） |
| 409 | 任务尚未完成（调用了 result/download 但任务仍在运行） |
| 500 | 服务端错误 |

**错误响应示例**:
```json
{
  "success": false,
  "message": "projectCode不能为空，必须指定项目编码",
  "data": null
}
```
