新增 GitLab 配置
API 模式：使用 GitLab REST API 获取数据，需配置 Personal Access Token。 适用于能够生成 Token 的场景。
GitLab API 根地址，例如 https://192.168.160.225/
在 GitLab 中生成：Settings → Access Tokens → 勾选 read_api 权限
克隆模式：通过 Git 协议直接克隆仓库获取提交记录，使用用户名+密码认证。 注意：GitLab CE v18.x 默认禁用密码 API 认证，建议使用此模式。
Git 仓库完整克隆地址，例如 https://192.168.160.225/pd-rs/code/dig-master.git，与 Jenkins 中配置的一致
